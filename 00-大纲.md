## 第一部分：入门篇 — 认识 Claude Code

### 第 1 章：Claude Code 是什么

- **1.1 从聊天机器人到 AI 智能体** — 一问一答 vs 自主干活
  - 番茄钟案例对比：ChatGPT（你来回复制粘贴） vs Claude Code（一句话出活）
- **1.2 Agent 的秘密：大模型循环（LLM Loop）**
  - 💡「简单来说」：大模型是大脑，Claude Code 是身体。大脑想→身体做→看结果→再想→再做，直到任务完成
  - 更完整的比喻：大模型 = 大脑 + 眼睛（感知环境、推理决策）；工具 = 手脚（读写文件、搜索网络、执行命令）；Agent = 大脑指挥手脚干活
  - 为什么 CC 特别聪明：它有「自主决策循环」— 感知环境 → 推理决策 → 调用工具 → 评估结果 → 继续循环，像一个成熟的职场人而不仅是实习生
  - 图示：LLM Loop 循环图（思考→选工具→执行→反馈→再思考）
- **1.3 Claude Code 和别的工具有什么不同** — 一张对比表
  - Copilot（代码补全）/ Cursor（付费 AI 编辑器）/ Trae（字节跳动，国内免费）/ Codex（OpenAI CLI）/ Windsurf
  - 💡「简单来说」：它们都用了好模型，但 Claude Code 的「驾驶系统」做得最好
- **1.4 为什么是「Harness 的集大成者」**
  - 同样的大脑、不同的身体 → 表现天差地别
  - 🔰「新手建议」：不用深究这个词，知道 Claude Code 不只是「套壳」就够了
- **1.5 它能做什么，不能做什么** — 五种核心提效场景 ⭐
  - ① Web Coding（用嘴写代码，描述需求 → CC 生成可运行文件）
  - ② 一次性复杂任务（周报 / 数据分析 / PPT 大纲）
  - ③ 自动化重复工作（Hooks「每次帮我检查格式」）
  - ④ 批量处理（改文件名 / 提取 PDF 关键信息 / 筛选简历 / 合并表格）
  - ⑤ 自我诊断（让 CC 帮你找出生活中最值得自动化的工作）
  - ❌ 不适合：需要实时网络爬虫、需要大量 GPU 计算的任务
  - 💡「简单来说」：凡是「读文件→思考→写文件→执行命令」的工作流，Claude Code 都能干
- **1.6 运行在本地，而不是云端**
  - 大脑在云端（调用 API），身体在本地（操作你的文件）

### 第 2 章：CLI 还是 Desktop？选择你的使用模式 ⭐

- **2.1 两种模式的核心差异** — 一张总览对比表
  - 同样的大脑，两种身体

| | CLI（命令行） | Desktop（桌面应用） |
|---|---|---|
| 界面 | 终端 TUI，纯文本 | GUI，文件树 + 多标签 |
| 图片 | 不能直接粘贴，需文件路径 | 拖拽 + Cmd+V 截图 |
| 第三方模型 | ✅ CC Switch | ❌ 仅 Claude 订阅 |
| 脚本自动化 | ✅ 管道、`claude -p`、cron | ❌ 不支持 |
| 上手难度 | 需要懂终端基本操作 | 🟢 零基础友好 |
| 适用人群 | 开发者、自动化场景 | 写作者、设计师、新手 |

- **2.2 CLI 模式详解** — 终端里的 AI 工作台
  - 独特优势：脚本化（`claude -p "任务"`）、管道输入、定时任务、第三方模型
  - 独特局限：无图片粘贴、无 GUI 文件树、键盘驱动
- **2.3 Desktop 模式详解** — 独立 App 体验
  - 独特优势：拖拽交互、截图粘贴、视觉化 diff、自动更新
  - 独特局限：绑定 Claude 订阅、不能自动化
- **2.4 怎么选：一张决策表** — 根据你的角色和任务选模式
- **2.5 两种模式可以同时用吗？** — 可以！同一账号，不同场景取长补短

### 第 3 章：快速安装与配置 ⭐ 重写

- **3.1 安装四步走（心里有数）** ⭐
  - ① 装 Node.js（CC 的运行环境）→ ② 装 Claude Code 本体 → ③ 搞定付费（官方订阅或第三方 API Key）→ ④ 配 Key 连上模型
  - 💡「简单来说」：跟装游戏一样——先装运行库，再装游戏，再登录账号，开玩
- **3.2 前置准备（两种模式通用）**
  - Node.js 安装：去官网下载对应系统版本 → 一路下一步
  - Git 是什么：🔰「新手建议」— 把它当成游戏的「存档系统」，不用会，让 AI 帮你操作
  - VS Code：文件树 + 终端 + 编辑器三合一
- **3.3 CLI 安装**
  - macOS：`brew install claude-code`（推荐）/ npm 一行命令
  - Windows：WSL + npm / Git Bash
  - 💡「简单来说」：装好终端里就能用 `claude` 命令
  - ⚠️「注意」：终端报错了别慌，截图发给任意 AI 都能帮你解决——看懂简单报错对 AI 来说太容易了
- **3.3 Desktop 安装**
  - macOS：从 claude.ai 下载 .dmg → 拖入 Applications → 登录 Claude 账号
  - 💡「简单来说」：跟装普通 Mac 软件一模一样
- **3.4 CLI 特别篇：中国用户配置第三方模型**
  - CC Switch 安装与配置
  - 接入 DeepSeek / Agnai / MiniMax / 豆包 等
  - `provider` 配置模板（可复制粘贴）
  - ⚠️「注意」：第三方模型功能可能受限（部分工具不支持）
- **3.5 Desktop 特别篇：订阅与账号**
  - 免费版 vs Pro（$20/月）vs Max（$100/月）的区别
  - Token 额度怎么看
  - 🔰「新手建议」：先用免费额度体验，够用再升级
- **3.6 验证安装**
  - CLI：`claude --version`
  - Desktop：打开 App → 第一次对话

### 第 4 章：界面与基本操作 ⭐ 分轨

- **4.1 CLI 界面详解**
  - 状态栏每条信息的意思（模型名 / Token 消耗 / 上下文占比 / 额度）
  - 思考面板（Thinking）— 看 AI 怎么想
  - 快捷键：Ctrl+C 中断、↑ 历史、Shift+Tab 切换权限
- **4.2 Desktop 界面详解**
  - 文件树侧边栏、对话面板、历史记录
  - @ 文件选择器、设置面板
  - 快捷键对照表
- **4.3 通用交互方式**
  - 打字对话（最常用）
  - @ 文件指定上下文
  - 斜杠命令（`/help`、`/clear`、`/compact` 等）
- **4.4 CLI 独有交互**
  - `claude -p "一句话任务"` → 一行命令
  - 管道：`cat error.log | claude -p "分析错误"`
  - 📋 常用一行命令模板
- **4.5 Desktop 独有交互**
  - 拖拽文件 / 图片到对话
  - Cmd+V 粘贴截图
- **4.6 提升交互效率的小技巧** ⭐
  - 🎤 **语音输入**：电脑自带语音 / 豆包输入法 / TypePlane，比打字快 3 倍，脑子一团乱麻时说给 AI 听反而更清晰
  - 📸 **截图沟通**：说不清楚就截图（Desktop Cmd+V / CLI 拖文件路径），一图胜千言
  - 🔄 **ESC 打断**：发现 CC 跑偏了立刻 ESC 打断，补充新想法继续
- **4.7 斜杠命令速查** — 按使用频率排序，标注 CLI / Desktop 通用性

### 第 6 章：版本控制 — 你的后悔药

- **6.1 为什么需要后悔药** — AI 改坏代码比你快，后悔药是必需品
- **6.2 Claude Code 内置回滚** — 双击 ESC / `/rewind`
- **6.3 内置回滚的局限** — 只能撤回文件，终端命令的副作用撤不回来
- **6.4 Git 才是真正的后悔药** — 💡「简单来说」：Git = 游戏存档系统。让 AI 帮你操作
  - 🔰「新手建议」：不用学 Git 命令，直接让 Claude Code 帮你 commit
- **6.5 实战：用 AI 管理版本** — 存档 → 大改 → 回滚
- **6.6 GitHub 远程备份** — 换电脑也能恢复

### 第 7 章：上下文管理

- **7.1 上下文是什么** — 每次对话、每个文件、每个操作结果都占空间
- **7.2 上下文满了会怎样** — 💡「简单来说」：AI 变笨了、忘事、抓不住重点
- **7.3 `/compact`：主动压缩** — 什么该压、什么不能压
- **7.4 `/clear`：清空重来**
- **7.5 `/context`：监控上下文** — CLI 状态栏实时显示 / Desktop 鼠标悬停
- **7.6 `/resume`：恢复对话** — CLI：`claude --continue` / Desktop：历史面板
- **7.7 省钱三招** ⭐
  - ① 任务完成后开新对话（不要一个窗口从选题聊到发布）
  - ② 用 CLAUDE.md 存项目背景（新对话自动读，不用重复介绍）
  - ③ 复杂任务先用 Plan Mode 规划（少返工就是省钱）
- **7.8 报错自救指南** ⭐
  - 报错了看不懂？→ 直接把报错信息复制给 CC，问「这是什么问题，怎么解决？」
  - CC 修不好还嘴硬？→ 跟它说「想想办法，上网搜一下找找哪里出错了」
  - 反复修不好？→ 新开对话，上下文太长了它会把自己绕进去
  - 实在搞不定？→ 截图发给另一个 AI（ChatGPT / Kimi），说「别人都能搞定，帮我看看」
- **7.9 最佳实践** — 任务粒度 / 定期压缩 / 善用子智能体
